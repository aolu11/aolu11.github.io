QQ/微信二次分享

2017年12月25日 分类 技术

前段时间做了微信的二次分享，其实比较简单，网上一搜一大把，就是引入微信的 jsapi，分别调用wx.onMenuShareAppMessage / wx.onMenuShareTimeline等方法即可

QQ 本质上也是一样的，但 QQ 的文档比微信更乱，后来搜了一下发现 QQ 除了用 js sdk 还可以通过<meta>标签来简单指定，感觉好像不用嵌 js sdk 了，于是：

<meta itemprop="name" content="这是分享的标题"/>
<meta itemprop="image" content="http://imgcache.qq.com/qqshow/ac/v4/global/logo.png" />
<meta name="description" itemprop="description" content="这是要分享的内容" />

这是官方文档里抽出来的（虽然不知是猴年马月没更新过的文档了），几个小问题：

  1. H5 标准，用了 itemprop 不能再用 name
  2. H5 标准，空标签无需再自闭合

…以上是扯蛋的，其实真正的问题在于，其它的都生效，但是标题，优先级比<title>低，这意味着只要网页有<title>，<meta>指定什么都没用，这有时候就蛋疼了，所以结论一下：

用<meta>能简单实现二次分享的内容控制，缺点在于优先级被<title>碾压，如果要求和<title>不一样，必须用 js sdk

对了…QQ 二次分享不像微信支持定制不同渠道的文案，不管分享到哪，都只有一套…（不管用<meta>还是 js sdk），不过对于微信朋友圈有个标题内容互换的选项…

http://open.mobile.qq.com/api/component/share

看这个页面就行了

手Q 限制，分享 URL 必须与页面 URL 同一域名，否则设置不生效
